Git-Repository-တစ်ခုကို Local-Storage-မှာ တည်ဆောက်ခြင်း

ဤလုပ်မှုမှာ Git-မူရင်းဌာနခွဲကို ပင်မဟု သတ်မှတ်ကာ ရှာဖွေပါမည်၊ git init command-အသုံးပြု၍ repository-ဖန်တီးနည်းနှင့် git pull command-ကို အသုံးပြုခြင်းဆိုင်ရာ တင်ပြခြင်း။

Git-သည် developer-များ အကြောင်းအရာများကို ခြေရာခံ၍ အရေးကြီးသော သိမ်းဆည်နိုင်စေမည့် ကိရိယာတစ်ခုဖြစ်သည်။ ၎င်းကို အသုံးပြု၍ အခြားသူများနှင့် ပူးပေါင်းကာ ကုဒ်များကို ဝေမျှနိုင်ခြင်း။ သင်၏ class repository-မှတဆင့် ကုဒ်ဝေမျှရာတွင် Git-ကို အသုံးပြုရပါလိမ့်မည်။ သင်ကုဒ်ကို GitHub-သိုလှောင်မှုဆီသို့ link-တစ်ခုကို ပေးပို့ရပါမည်။

Local-မှာ Git Default Branch-ကို main-အဖြစ် သတ်မှတ်ပါ။

အရင်ဦးဆုံး Local Git-၏ အကိုင်းအခက်ကို main-လို့ သတ်မှတ်ဖို့ လိုအပ်ပါသည်။

ရှေးယခင်ကဆိုရင် Codebase-တစ်ခု၏ အဓိကကိုယ်ထည်အဖြစ် master-ဆိုတဲ့ အမည်နဲ့ဟာကို အများဆုံးအသုံးပြုခဲ့သည်။ အသုံးရာတွင် လူကြိုက်အများဆုံးဖြစ်လာခဲ့သည်။ ယခုအခါမှာတော့ GitHub-သည် သင်တန်းတွင်ရှိ ပရောဂျက်များကိုပင် သိုလှောင်ရာနေရာအဖြစ် main-ဆိုတဲ့ ပင်မကို အသုံးပြုလာကြသည်။

GitHub-သည် မူရင်းဌာနခွဲဆိုင်ရာကို ပြောင်းလဲပါသော်လည်း စက်အတွင်းစက်သည် master-ဌာနခွဲကိုပင် အသုံးပြုကာ ပရောဂျက်များကို စတင်လုပ်ဆောင်နေဆဲဖြစ်သည်။ ဒါကြောင့် သင်သည် ၎င်းmaster-ကို main-ပင်မသို့ ပြောင်းလဲရန် လိုအပ်ပါသည်။

သင့်စက်အတွင်းတွင် ထည့်သွင်းထားသော Git Version-ကို စစ်ဆေးရန် conmmand line-တွင် "git --version"-ကို ရိုက်ထည့်စစ်ဆေးကြည့်ပါ။ သင့်ရဲ့ Git Version-ကို 2.28-နှင့် အထက်ဖြစ်နေသေးပါက Git-ကို update-လုပ်ရန် လိုအပ်ပါသည်။

** Local Repository တစ်ခုကို စတင်လိုက်ပါ

GitHub-မှာ သိုလှောင်မှုတစ်ခုကို ဖန်တီးပြီး စက်အတွင်းတွင် ပုံတူပွားနည်းကို လေ့လာခဲ့ပြီးဖြစ်သည်၊ ဒီတစ်ခါ "git init" command-ကို အသုံးပြုပြီး စက်အတွင်းကနေ repository-အသစ်တစ်ခုကို စတင်ကြလိုက်ရအောင်၊ git init-ကို အသုံးပြုသဖြင့် ရှိပြီးသားပရောဂျက်များကို Git Repository-အဖြစ်သို့ လွယ်ကူစွာ ပြောင်းနိုင်စေပါသည်။
စက်အတွင်းတွင် "git-init-sample" -ဟု ပရောဂျက်အမည်ပေးရန် "mkdir git-init-sample" command line-တွင် လမ်းညွှန်အသစ်တစ်ခုကို ဖန်တီးဖို့ စတင်ရအောင်၊

ထိုနောက် တည်ဆောက်ထားတဲ့ လမ်းညွှန်အသစ်ဆီ သွားဖို့ရန် ဥပမာ"cd git-init-sample" cd-ခံ၍ သွားပါ၊ file-တစ်ခုတည်ဆောက်ရန် "touch" command-ခံ၍ ဥပမာ"touch index.html"-ဖိုင်တစ်ခု ထည့်ပါ၊ "git-init-sample/index.html"ပရောဂျက်ကို Git Repository-အဖြစ်သို့ ရောက်ရန် "git init"ကို ပြုလုပ်သောအခါ လမ်းညွှန်ချက်သည် ပရောဂျက်ဖိုင်တွဲတွင် ရှိနေရပါမည်။

"git init"-သည် သိုလှောင်မှုဖိုင်တွဲများ ပါဝင်သော .git -အမည်ရှိလမ်းညွှန်အခွဲတစ်ခုပါ။ ပရောဂျက်ထဲရှိဖိုင်တွဲများကို ဗားရှင်းထိန်းချုပ်ခြင်း, စတင်ခြေရာခံခြင်းကို ကတိပြုရန် လိုအပ်ပါသည်။ ဖိုင်တွေရဲ့အခြေနေကို စစ်ဆေးရန် အရင်ဦးဆုံး "git status"ကို အသုံးပြုပါ။ ဖိုင်တွေရဲ့အခြေနေကို ခြေရာခံဖို့ရန် "git add ."ကို အသုံးပြုပါ။ ဖိုင်တွေရဲ့အခြေနေကို ခြေရာခံပြီးနောက် ကတိပြုရန်အတွက် -git commit -m "initial commit"- ဖြင့် စက်အတွင်းသိုလှောင်မှုကို GitHub-အဝေးထိန်းသိုလှောင်မှုဆီသို့ ချိတ်ဆက်ဖို့ အဆင်သင့်ပါပဲ။

** GitHub.com ကို သွားပါ၊ New Repository-မှာ Local-တုန်းက ပေးခဲ့တဲ့ ပရောဂျက်နာမည်အတိုင်း နာမည်တူဖို့ အရေးကြီး နာမည်ပေးပါ၊ "Create Repositoy"ကို နှိပ်ပါ၊ ခေါင်းစဉ်အောက်တွင်ရှိ Link-ထဲက "HTTPS/SSH URL"ထဲရှိ .git-နဲ့ဆုံးတဲ့ ကုဒ်ကို ကူးယူပါ။ ထိုနောက် Local command line-တွင် git remote add origin " -- "ကွက်လပ်ထဲတွင် ကူးယူလာသော ကုဒ်ကို ထည့်ပါ။ "git branch -M main" "git push -u origin main"ကို အသုံးပြုပြီး "git push"ခြင်းကို အောင်မြင်ပါလိမ့်မည်။

== Git pull==ခြင်း အဝေးရှိသိုလှောင်မှုဖြစ်တဲ့ GitHub repository-ထဲ ပြုပြင်ပြောင်းလဲထားမှုတွေကို Local repository-ထဲ ရောက်လာအောင် "git pull origin main"ကို command line-အသုံးပြု၍ ရယူလိုက်ပါ။ ပြောင်းလဲသောဖိုင်ကို VS Code-တွင် ပြန်လည်စစ်ဆေးနိုင်ပါသည်။
